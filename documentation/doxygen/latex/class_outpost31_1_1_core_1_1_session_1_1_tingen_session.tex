The Tingen \doxylink{namespace_outpost31_1_1_core_1_1_session}{Session} contains all of the information that Tingen needs to do what it does, including\+: 
\begin{DoxyItemize}
\item \doxylink{namespace_outpost31_1_1_core_1_1_configuration}{Configuration} settings (user-\/definable settings from the {\itshape config\+File} ) 
\item Static settings (these do not change between sessions) 
\item Runtime settings (unique to the current session) 
\item Data sent from \doxylink{namespace_outpost31_1_1_core_1_1_avatar}{Avatar}, including the {\itshape sent\+Option\+Object}  and {\itshape sent\+Script\+Parameter}  
\item Data derived from the {\itshape sent\+Option\+Object}  or {\itshape sent\+Script\+Parameter} . 
\item Required Tingen \doxylink{namespace_outpost31_1_1_module}{Module} details. 
\end{DoxyItemize}

{\bfseries{More information about this method\+:}}~\newline
 This method is one of the most important in Tingen, as it builds the Tingen \doxylink{namespace_outpost31_1_1_core_1_1_session}{Session} object.~\newline
~\newline
 The \doxylink{namespace_outpost31_1_1_core_1_1_session}{Session} object is built in this order\+: 
\begin{DoxyEnumerate}
\item A basic \doxylink{class_outpost31_1_1_core_1_1_session_1_1_tingen_session}{Tingen\+Session} object is initalized with the current date, time, configuration settings, and \doxylink{namespace_outpost31_1_1_core_1_1_avatar}{Avatar} data. 
\item \doxylink{namespace_outpost31_1_1_core_1_1_framework}{Framework} information is added seperately. 
\item Trace log information is added seperately. 
\item \doxylink{namespace_outpost31_1_1_module}{Module} objects are added seperately. 
\end{DoxyEnumerate}
